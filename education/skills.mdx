## Software Architecture & Design

### Design Patterns

<Accordion title="Common Patterns">
**Creational Patterns:**
- Singleton: Single instance management
- Factory: Object creation abstraction
- Builder: Complex object construction

**Structural Patterns:**
- Adapter: Interface compatibility
- Decorator: Dynamic behavior extension
- Facade: Simplified interface to complex systems

**Behavioral Patterns:**
- Observer: Event notification system
- Strategy: Algorithm encapsulation
- MVC/MVVM: Separation of concerns
</Accordion>

### Architectural Patterns

- **Model-View-Controller (MVC)**: Traditional separation of concerns
- **Model-View-ViewModel (MVVM)**: Modern approach for reactive UIs
- **Clean Architecture**: Dependency rule and layer separation
- **Microservices**: Distributed system architecture
- **RESTful API Design**: Resource-based web services
